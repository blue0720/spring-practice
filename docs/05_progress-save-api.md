# 05. 進行状況の保存・読込API

対応フェーズ: フェーズ4

## 概要
ログインユーザーごとに1つの進行状況(セーブデータ)をDBに永続化するAPIを実装する。

## Todo
- [ ] `GET /api/progress`: 該当ユーザーの進行状況取得(`ROLE_PLAYER`以上)
- [ ] `POST /api/progress`: 進行状況の保存(Ajax/fetch経由、`ROLE_PLAYER`以上)
- [ ] 保存内容の設計: 現在の章、区画ごとの交渉済みフラグ、最終交渉の結果など
- [ ] `GET /game` アクセス時にDBから進行状況を読み込み、Thymeleafで初期状態として埋め込む(またはJS側からAPI取得)
- [ ] 交渉パズル1件完了 or マップ帰還タイミングで `POST /api/progress` を呼び出すフロント側実装
- [ ] `CompletedEncounter` への交渉履歴記録(誰といつどんな結果で交渉したか)
- [ ] 複数セーブスロットは非対応であることの確認(スコープ外)
